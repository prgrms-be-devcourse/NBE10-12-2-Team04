# TripTrace
```
TripTrace는 여행 단위로 일정과 기록을 묶어 관리하는 여행 기록 서비스입니다.
사용자는 여행기를 생성하고, 해당 여행기 안에서 날짜별 게시물을 작성할 수 있습니다.

사용자가 최대 20장의 사진을 업로드하면 사진의 촬영일과 위치 메타데이터를 분석해 여행 기록을 자동으로 분리하여 게시물과 마커를 생성할 수 있도록 확장하고 있습니다.
각 게시물은 여행 중 특정 순간의 기록을 담으며, 위치 기반 마커와 이미지를 연결해 장소와 사진 중심으로 여행 경험을 정리할 수 있습니다.
공개 여행기는 다른 사용자가 조회할 수 있으며, 좋아요 등록/취소에 따라 여행기의 좋아요 수를 자동으로 반영해 인기 여행기를 확인할 수 있도록 확장하고 있습니다.
```
## 주요 기능

### 여행기 관리

- 여행기 생성, 조회, 수정, 삭제
- 공개/비공개 여행기 관리
- 내 여행기 목록 조회
- 공개 여행기 목록 조회

### 게시물 관리

- 여행기별 게시물 생성, 조회, 수정, 삭제
- 여행 날짜 기준 게시물 관리
- 공개/비공개 여행기에 따른 접근 제어

### 좋아요 기능

- 여행기 좋아요 등록
- 여행기 좋아요 취소
- 좋아요 여부 조회
- 좋아요 등록/취소 시 여행기 좋아요 수 자동 반영

### 마커 기능

- 게시물별 대표 마커 생성
- 마커 조회, 수정, 삭제
- GPS 좌표, 장소명, 방문 시간 관리
- 수동/자동 생성 마커 구분
- 이미지 메타데이터 기반 자동 마커 생성 확장 고려

### 이미지 기능

- 최대 20장 사진 업로드 기반 여행 기록 자동 분리
- 이미지 정보 관리
- 원본 이미지 URL 및 썸네일 URL 관리
- 파일 크기, MIME 타입 저장
- GPS, 촬영일, 기기 정보 등 이미지 메타데이터 추출
- 이미지 방향 보정 및 썸네일 생성 확장 고려

- TypeScript
- pnpm

- Git Hooks를 통한 브랜치명/커밋 메시지 규칙 관리
- GitHub Actions 기반 Backend CI
